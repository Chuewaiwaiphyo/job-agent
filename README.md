# Semantic Job Recommendation System

A Retrieval-Augmented Generation (RAG)-inspired job recommendation system that uses semantic search to match user career profiles with relevant job descriptions. The project compares traditional information retrieval techniques (TF-IDF and BM25) against a dense vector retrieval approach using Sentence Transformers and FAISS.

---

## Project Overview

Traditional job recommendation systems rely heavily on keyword matching. However, job seekers often describe their skills, experience, and career goals using natural language rather than explicit job titles or predefined keywords.

This project addresses this limitation by implementing a semantic retrieval pipeline capable of understanding the meaning behind a user's profile and retrieving relevant jobs even when exact keywords are absent.

The system uses sentence embeddings generated by a pre-trained transformer model and performs similarity search using FAISS to identify the most relevant job opportunities.

## Retrieval Methods

### Baseline 1: TF-IDF

TF-IDF converts documents into sparse vectors and ranks them using cosine similarity.

Configuration:

```python
TfidfVectorizer(
    stop_words="english",
    ngram_range=(1,2),
    max_features=10000
)
```

---

### Baseline 2: BM25

BM25 is a probabilistic ranking algorithm commonly used in information retrieval systems.

Tokenization:

```python
re.findall(r"\b\w+\b", text.lower())
```

Library:

```python
rank-bm25
```

---

### Proposed Method: Sentence Embeddings + FAISS

The proposed semantic retrieval system uses transformer-based embeddings and vector similarity search.

Embedding Model:

```text
sentence-transformers/all-MiniLM-L6-v2
```

Vector Dimension:

```text
384
```

Similarity Metric:

```text
Cosine Similarity
```

FAISS Index:

```python
faiss.IndexFlatIP()
```

Embeddings are normalized before indexing to enable cosine similarity search.

## Evaluation

The project compares lexical retrieval methods against semantic retrieval.

| Method                      | Relevance |
| --------------------------- | --------- |
| TF-IDF                      | Low       |
| BM25                        | Low       |
| FAISS + SentenceTransformer | High      |

### Analysis

TF-IDF and BM25 depend primarily on keyword overlap between queries and documents. As a result, they struggle when users describe their experience using natural language rather than explicit skill keywords.

The FAISS-based semantic retrieval approach successfully captures contextual meaning and semantic relationships. It can associate concepts such as:

* "learn from data" → Machine Learning
* "understand human language" → NLP
* "predictive solutions" → Data Science

This enables significantly more relevant job recommendations.
